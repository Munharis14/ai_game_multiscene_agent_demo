# Demo 测试记录：游戏研发多场景 AI Agent

## 1. 测试目标

验证 Demo 是否能够完成多场景 Agent 链路：接收问题、判断任务类型、调用对应工具、返回工具结果，并展示执行过程。覆盖研发知识检索、策划活动规则检查和运营玩家反馈摘要三个场景。

## 2. 测试环境

| 项目 | 说明 |
| --- | --- |
| 运行方式 | Streamlit 本地运行 |
| 地址 | http://127.0.0.1:8700 |
| 技术栈 | Python、Streamlit、LangChain、Chroma |
| 知识库 | docs 目录下的 Markdown 游戏资料 |
| 反馈样本 | data/player_feedback.md |
| 模型状态 | 支持 OpenAI-compatible LLM；模型不可用时使用本地检索摘要兜底 |

## 3. 测试用例

| 编号 | 测试问题 | 期望工具 | 验证重点 | 结果 |
| --- | --- | --- | --- | --- |
| 1 | 新手任务第一步是什么？ | search_game_docs_tool | 能否检索新手流程资料 | 通过 |
| 2 | 夏日星潮活动的参与条件是什么？ | search_game_docs_tool | 能否定位活动规则和客服 FAQ | 通过 |
| 3 | 检查夏日星潮活动规则是否完整 | check_activity_rule_tool | 能否输出活动规则字段检查表 | 通过 |
| 4 | 帮我总结一下玩家反馈里的主要问题 | summarize_player_feedback_tool | 能否输出高频问题、情绪判断和处理建议 | 通过 |
| 5 | 充值不到账客服应该怎么处理？ | search_game_docs_tool | 能否支撑客服知识库问答 | 通过 |

## 4. 当前结论

当前 Demo 已经可以证明多场景 AI Agent 的最小可行性：游戏研发资料可以被接入知识库，玩家反馈可以作为运营分析样本，用户通过自然语言提问后，Agent 可以选择不同工具完成知识检索、规则检查和反馈摘要。

## 5. 后续优化

- 接通稳定可用的大模型额度，展示完整 LLM 生成回答。
- 增加更多工具，例如测试日志分析、配置表检查和客服工单分流。
- 增加回答评分和用户反馈按钮，用于后续效果评估。
- 增加权限控制和数据脱敏机制，避免真实项目资料外泄。
