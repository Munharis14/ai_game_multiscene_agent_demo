# Demo 测试记录：游戏研发文档问答 Agent

## 1. 测试目标

验证 Demo 是否能够完成最小 Agent/RAG 链路：接收问题、调用文档检索工具、返回相关片段、生成回答或兜底摘要，并展示引用来源。

## 2. 测试环境

| 项目 | 说明 |
| --- | --- |
| 运行方式 | Streamlit 本地运行 |
| 地址 | http://127.0.0.1:8700 |
| 技术栈 | Python、Streamlit、LangChain、Chroma |
| 知识库 | docs 目录下的 Markdown 游戏资料 |
| 模型状态 | 支持 OpenAI-compatible LLM；模型不可用时使用本地检索摘要兜底 |

## 3. 测试用例

| 编号 | 测试问题 | 期望命中文档 | 验证重点 | 结果 |
| --- | --- | --- | --- | --- |
| 1 | 新手任务第一步是什么？ | 新手任务流程.md | 能否检索新手流程资料 | 通过 |
| 2 | 夏日星潮活动的参与条件是什么？ | 活动规则说明.md、版本更新公告.md | 能否定位活动规则 | 通过 |
| 3 | 充值不到账客服应该怎么处理？ | 客服FAQ.md | 能否支撑客服知识库问答 | 通过 |
| 4 | 背包满了奖励会丢失吗？ | 道具与奖励规则.md、客服FAQ.md | 能否回答道具和奖励规则 | 通过 |
| 5 | 本次版本新增了哪些内容？ | 版本更新公告.md | 能否检索版本公告 | 通过 |

## 4. 当前结论

当前 Demo 已经可以证明文档问答 Agent 的最小可行性：游戏研发资料可以被接入知识库，用户通过自然语言提问后，Agent 可以调用检索工具找到相关资料，并展示答案与引用来源。

## 5. 后续优化

- 接通稳定可用的大模型额度，展示完整 LLM 生成回答。
- 增加更多工具，例如测试日志分析、配置表检查和玩家反馈摘要。
- 增加回答评分和用户反馈按钮，用于后续效果评估。
- 增加权限控制和数据脱敏机制，避免真实项目资料外泄。
